PALM-mGWAS Usage Guide

This page explains how to run the PALM-mGWAS R package: key functions, required inputs, outputs, and visualization workflows.

Workflow at a Glance

Info
Info.R creates SNP, feature, and sequencing-depth summaries.
Step0 Check
step0_checkInput.R checks IID consistency across abundance, covariates, and PLINK .fam, and reorders tables to match genotype order.
Step1 Null
step1_null.R fits the PALM null model from abundance (and optional covariates).
Step2 Summary
step2_summary.R reads PLINK genotypes and writes per-phenotype estimates.
Step3 Meta
step3_meta.R meta-analyzes multiple studies and adds meta_* columns.
Visualization
Visualization.R generates Manhattan and forest plots.

Environment & Dependencies

  • R ≥ 4.2; use a clean renv/conda environment if possible.
  • Main deps: PALM, snpStats, data.table, dplyr, ggplot2, qqman, stringr, tibble.
  • Install example: install.packages(c("PALM","snpStats","data.table","dplyr","ggplot2","qqman","stringr","tibble"))

Installation (two options)

Option A: pixi (recommended for local runs)

# 1) install pixi (if not yet)
curl -fsSL https://pixi.sh/install.sh | bash
# 2) set PATH if needed (depends on installer hint)
# 3) in repo root, create env and fetch deps
pixi install
# 4) run workflows with pixi run ... (see steps below)

Option B: Docker image

# build (from repo root)
docker build -t palm-mgwas .

# run interactively, mounting your data
docker run --rm -it -v "$PWD":/work -w /work palm-mgwas bash

# inside the container, call packaged scripts (pixi already available)
pixi run --manifest-path=/app/pixi.toml Rscript /app/extdata/step1_null.R --help

The Dockerfile installs pixi, PALM, PLINK1.9, and wrapper scripts under /app/extdata/; mount your data into the container and reference it via absolute paths (e.g., /work/...). Info and visualization scripts can be run in any order depending on your pipeline.

Info

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/Info.R \
    --genoFile=${genoPrefix} \
    --abdFile=${abdFile} \
    --outputSnpFile=${outputSnpFile} \
    --outputFeatureFile=${outputFeatureFile} \
    --outputSeqDepthFile=${outputSeqDepthFile}

Run the packaged CLI; no need to call exported R functions directly.

  • Purpose: quick QC summaries before modeling.
  • Inputs: PLINK prefix (.bed/.bim/.fam) and abundance TSV (sample IDs in column 1).
  • Outputs: snp_info.tsv (CHR, SNP, POS, A1, A2, N, AF); feature_info.tsv (FeatureID, Prevalence, AvgProportion); seqdepth.tsv (SampleID, SeqDepth).
  • QC to check: AF within 0–1, prevalence not all zero, sequencing-depth distribution reasonable.
  • Typical use: filter rare features or low-depth samples and annotate downstream plots.
  • Key tunables: none in code; filtering thresholds (e.g., prevalence, AF, depth) are user-defined after these tables are produced.

Step0: Check and Align Input IDs

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step0_checkInput.R \
    --abdFile=${abdFile} \
    --covFile=${covFile} \
    --genoFile=${genoPrefix}
  • Purpose: verify that abundance, covariates, and PLINK .fam contain the same IID set before modeling.
  • Behavior: if IID sets match but row order differs, the script rewrites abdFile and covFile in PLINK .fam order.
  • Requirement: all three inputs are mandatory in the current CLI.
  • Failure cases: duplicated IDs, missing values, missing .fam, or non-identical IID sets.

Step1: Fit Null Model

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step1_null.R \
    --abdFile=${abdFile} \
    --covFile=${covFile} \
    --outputPrefix=${palm1_step1_prefix}
  • Inputs: abundance table; optional covariate table (same sample IDs/order as abundance).
  • Outputs: outputPrefix.rda containing modglmm; directories are created if missing.
  • Behavior: pass --covFile=NULL to run covariate-free; the CLI converts empty string or NULL to R NULL.
  • Tips: ensure no duplicated sample IDs; use relative abundance; keep sessionInfo for reproducibility.
  • Key args: abdFile, covFile, outputPrefix.

Step2: Per-study Summary

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step2_summary.R \
    --inFile=${genoPrefix} \
    --NULLmodelFile=${palm1_step1_prefix}.rda \
    --PALMOutputFile=${palm1_step2_prefix} \
    --chrom=${chrom} \
    --correct=${correct} \
    --useCluster=${useCluster}
  • Inputs: PLINK prefix (.bed/.bim/.fam), null model .rda.
  • Outputs: per-phenotype TSVs named PREFIX_.txt; columns SNP, CHR, POS, est, stderr, pval.
  • SNP format: dots are replaced by colons, e.g., chr1:12345:REF:ALT.
  • Chrom filter: current CLI default is empty string; use --chrom=1 or --chrom=chr1 to restrict, otherwise leave empty to keep all chromosomes.
  • Clustering: if useCluster = TRUE, FID from .fam is passed to PALM; automatically disabled when all FIDs are 0.
  • correct: forwarded to PALM::palm.get.summary(); the CLI converts NULL or empty string to R NULL.
  • Key args: inFile, NULLmodelFile, PALMOutputFile, chrom, correct, useCluster.

Step3: Meta Analysis

# studyDirList TSV lines: studyIDdir (e.g., "study1\t${outputFolder}/study1")
pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step3_meta.R \
    --studyDirList=${studyFile} \
    --inPrefix=${metaPrefix} \
    --outDir=${metaDir} \
    --outPrefix=${metaPrefix} \
    --keepHet=TRUE \
    --metaMethod=EE \
    --featuresFile=NULL
  • Inputs: studyDirList is a text file with two columns per line: studyID and Step2 output directory. Files inside those directories must match inPrefix + <pheno> + ".txt".
  • Outputs: one file per phenotype in outDir named outPrefix<pheno>.txt, containing meta_est, meta_stderr, meta_pval, optional meta_pval.het, plus per-study study*_est/study*_stderr.
  • Single-study mode: still useful to standardize column names for plotting.
  • Heterogeneity: set --keepHet=FALSE to drop the het column and mimic 6-column Step2 format.
  • Meta method: --metaMethod is passed to metafor::rma(); default is EE.
  • Feature subset: --featuresFile is optional and should contain one phenotype per line; use NULL or omit content to infer features automatically.
  • Key args: studyDirList, inPrefix, outDir, outPrefix, keepHet, metaMethod, featuresFile.

Visualization

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/Visualization.R \
    --metaDir=${metaDir} \
    --pattern=${pattern} \
    --plotDir=${plotDir} \
    --pheno=${pheno:-NA} \
    --snp=${snp:-NA} \
    --out=${outFile:-NA} \
    --pCut=${pCut} \
    --sigOnlyPheno=FALSE \
    --printCut=1e-8 \
    --showMeta=${showMeta} \
    --showHet=${showHet} \
    --width=${plotWidth} \
    --height=${plotHeight}
  • Case 1: no --pheno and no --snp gives a combined overview across phenotypes and writes combined_hits.png.
  • Case 2: --pheno only gives a phenotype-specific Manhattan plot and QQ plot.
  • Case 3: --snp only gives a forest plot for one SNP across all phenotypes found in the input files.
  • Case 4: both --pheno and --snp give a forest plot for one phenotype-SNP pair across studies.
  • Outputs: plots saved to --plotDir; if --out is not provided, the script auto-generates a file name based on mode, phenotype, and SNP.
  • Input scope: --metaDir can point to Step3 outputs or any directory of Step2/Step3-like files matching --pattern.
  • Key args: metaDir, plotDir, pattern, pheno, snp, out, pCut, sigOnlyPheno, printCut, showMeta, showHet, width, height.

Example Workflow: from input to output

The same example data can be read as a concrete workflow: what goes in, which script is called, which parameters matter, and what files come out.

Input: example/input/study1/abd.txt and PLINK files example/input/study1/geno.bed/.bim/.fam.

2. Step0 + Step1: aligned IDs to null model

Input: example/input/study1/abd.txt, example/input/study1/cov.txt, and example/input/study1/geno.fam.

# Step0 checks that IID sets match and reorders abd/cov if needed
pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step0_checkInput.R \
    --abdFile=${WORK}/example/input/study1/abd.txt \
    --covFile=${WORK}/example/input/study1/cov.txt \
    --genoFile=${WORK}/example/input/study1/geno

# Step1 fits the null model used by association testing
pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step1_null.R \
    --abdFile=${WORK}/example/input/study1/abd.txt \
    --covFile=${WORK}/example/input/study1/cov.txt \
    --outputPrefix=${WORK}/example/output/study1/step1_allpheno

Output: example/output/study1/step1_allpheno.rda, which is then passed into Step2 as --NULLmodelFile.

3. Step2: null model + genotype to per-phenotype association files

Input: PLINK prefix example/input/study1/geno and null model example/output/study1/step1_allpheno.rda.

pixi run --manifest-path=${WORK}/pixi.toml Rscript ${WORK}/extdata/step2_summary.R \
    --inFile=${WORK}/example/input/study1/geno \

    --PALMOutputFile=${WORK}/example/output/study1/step2_allchr \

    --correct=NULL \
    --useCluster=TRUE

Output: one file per phenotype, such as example/output/study1/step2_allchr_g1.txt, example/output/study1/step2_allchr_g2.txt, and so on.

Inputs/Outputs Cheat Sheet

  • Inputs: abundance table, covariates for current Step0 CLI, and PLINK genotype prefix.
  • Intermediates: Step0 may rewrite abdFile/covFile; Step1 writes *.rda; Step2 writes per-phenotype files.
  • Outputs: meta result tables (meta_* columns), Manhattan/QQ/forest JPEGs, optional top lists.

Troubleshooting

  • IID mismatch: run step0_checkInput.R first; it requires identical IID sets across abundance, covariates, and PLINK .fam.
  • “modglmm not found”: ensure the .rda from fitNULL() is correctly passed to Step2.
  • Missing columns: Step2/3 files must contain SNP, CHR, POS, est, stderr, pval; meta files need meta_pval.
  • FID all zero: getSummary() auto-disables clustering; fill FID in .fam if clusters are needed.

See workflow/ and example/ for batch scripts or sample data to pair with this guide.
